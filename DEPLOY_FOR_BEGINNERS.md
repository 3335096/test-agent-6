# 🚀 Деплой Мастер-агента на Railway — Инструкция для начинающих

Пошаговое руководство без сложных терминов. Следуйте пунктам по порядку.

---

## 📋 Что понадобится

1. **Компьютер** с Windows/Mac/Linux
2. **Браузер** (Chrome, Firefox, Safari)
3. **API ключ OpenRouter** (получим на шаге 1)
4. **Аккаунт GitHub** (создадим на шаге 2)
5. **Аккаунт Railway** (создадим на шаге 4)

---

## ШАГ 1: Получаем API ключ для ИИ 🤖

Этот ключ нужен, чтобы агент мог общаться с искусственным интеллектом.

### 1.1 Регистрация на OpenRouter

1. Откройте сайт: https://openrouter.ai
2. Нажмите кнопку **"Sign In"** (войти) в правом верхнем углу
3. Выберите **"Sign up with Google"** (регистрация через Google) — это проще всего
4. Выберите свою Google-учётную запись
5. Готово! Вы вошли в систему

### 1.2 Создаём API ключ

1. На сайте OpenRouter нажмите на **"Keys"** в меню сверху
   - Или перейдите сразу: https://openrouter.ai/keys
2. Нажмите большую кнопку **"Create Key"** (создать ключ)
3. В поле **"Name"** напишите: `Master Agent service.by`
4. Нажмите **"Create"**
5. **ВАЖНО!** Появится длинный текст начинающийся на `sk-or-v1-...`
6. Скопируйте этот текст и **сохраните в блокнот/файл** — он показывается только один раз!

> 💡 **Совет:** Создайте текстовый файл на рабочем столе с названием `api-key.txt` и вставьте туда ключ. Он понадобится на шаге 5.

## ШАГ 2: Создаём аккаунт на GitHub 🐙

GitHub — это сайт где хранится код. Railway будет брать код оттуда.

### 2.1 Регистрация

1. Откройте: https://github.com
2. Нажмите зелёную кнопку **"Sign up"** (регистрация)
3. Введите:
   - **Email** (ваш email)
   - **Password** (придумайте пароль, минимум 8 символов)
   - **Username** (имя пользователя, например `ivan-master-agent`)
4. Нажмите **"Create account"**
5. Подтвердите email — зайдите в почту и нажмите на ссылку от GitHub
6. Ответьте на пару вопросов (можно пропустить, нажав "Skip")

### 2.2 Создаём репозиторий (хранилище кода)

1. На главной странице GitHub нажмите зелёную кнопку **"Create repository"**
   - Или перейдите: https://github.com/new
2. В поле **"Repository name"** напишите: `master-agent-serviceby`
3. В поле **"Description"** (описание) напишите: `Мастер-агент для Telegram-канала service.by`
4. Оставьте галочку **"Public"** (публичный) — так проще
5. **СНИМИТЕ** галочку "Add a README file" (уберите её)
6. Нажмите зелёную кнопку **"Create repository"**

**Сохраните открывшуюся страницу — она понадобится на шаге 3!**

## ШАГ 3: Загружаем код на GitHub 📤

### 3.1 Скачиваем готовый проект

Проект уже собран и лежит здесь: `/mnt/okcomputer/output/app/`

Вам нужно:
1. Скопировать все файлы из этой папки на свой компьютер
2. Загрузить их на GitHub

**Вариант А — через GitHub Desktop (проще):**

1. Скачайте GitHub Desktop: https://desktop.github.com
2. Установите и войдите в свой аккаунт
3. Нажмите **"File" → "Clone repository"**
4. Выберите свой репозиторий `master-agent-serviceby`
5. Выберите папку на компьютере куда сохранить
6. Скопируйте файлы из `/mnt/okcomputer/output/app/` в эту папку
7. В GitHub Desktop напишите комментарий: "Initial commit"
8. Нажмите **"Commit to main"**
9. Нажмите **"Push origin"**

**Вариант Б — через браузер (ещё проще, но дольше):**

1. Откройте свой репозиторий на GitHub
2. Нажмите **"Add file" → "Upload files"**
3. Перетащите ВСЕ файлы из папки `/mnt/okcomputer/output/app/` (кроме `node_modules`)
4. Прокрутите вниз, напишите: "Initial commit"
5. Нажмите **"Commit changes"**

> ⚠️ **Важно:** Не загружайте папку `node_modules` — она очень большая и не нужна!

## ШАГ 4: Регистрация на Railway 🚂

Railway — это сервис который запустит ваш агент в интернете.

### 4.1 Создаём аккаунт

1. Откройте: https://railway.app
2. Нажмите **"Start for Free"** (начать бесплатно)
3. Нажмите **"Login with GitHub"** (войти через GitHub)
4. Разрешите доступ — нажмите зелёную кнопку **"Authorize Railway"**
5. Заполните профиль (можно пропустить):
   - **Name** — ваше имя
   - **Company** — можно пропустить
6. Нажмите **"Complete"** (завершить)

### 4.2 Подтверждаем email

1. Railway пришлёт письмо на ваш email
2. Откройте почту и нажмите **"Verify Email"**
3. Готово!

---

## ШАГ 5: Создаём проект на Railway 🛠️

### 5.1 Создание проекта

1. На главной странице Railway нажмите **"New Project"** (новый проект)
2. Выберите **"Deploy from GitHub repo"** (развернуть из GitHub)
3. Нажмите **"Configure GitHub App"** (настроить GitHub)
4. Выберите **"Only select repositories"** (только выбранные репозитории)
5. Нажмите на поле и выберите свой репозиторий `master-agent-serviceby`
6. Нажмите зелёную кнопку **"Install & Authorize"**
7. Railway покажет список репозиториев — выберите `master-agent-serviceby`
8. Нажмите **"Add Variables"** (добавить переменные)

### 5.2 Добавляем секретные настройки (ПЕРЕМЕННЫЕ)

Это самый важный шаг! Здесь мы укажем API ключ.

1. Нажмите кнопку **"New Variable"** (новая переменная)
2. В поле **KEY** напишите: `OPENROUTER_API_KEY`
3. В поле **VALUE** вставьте ваш ключ из шага 1 (начинается на `sk-or-v1-`)
4. Нажмите **"Add"**

Добавьте ещё две переменные:

**Вторая переменная:**
- KEY: `OPENROUTER_MODEL`
- VALUE: `anthropic/claude-3.5-sonnet`

**Третья переменная:**
- KEY: `NODE_ENV`
- VALUE: `production`

В итоге должно быть 3 переменные:
- `OPENROUTER_API_KEY` = ваш ключ
- `OPENROUTER_MODEL` = anthropic/claude-3.5-sonnet
- `NODE_ENV` = production

5. Нажмите **"Deploy"** (развернуть)

## ШАГ 6: Ждём деплой ⏳

Railway начал собирать и запускать ваш проект.

### 6.1 Что происходит

1. Railway скачивает код с GitHub
2. Устанавливает все зависимости (npm install)
3. Собирает фронтенд (npm run build)
4. Запускает сервер

### 6.2 Сколько ждать

- Обычно **3-5 минут**
- Можно смотреть процесс в разделе **"Deployments"**

### 6.3 Как понять что готово

1. Перейдите в раздел **"Deployments"** (слева в меню)
2. Найдите последний деплой
3. Статус должен быть **"Success"** (зелёная галочка ✅)

Если видите красный крестик ❌ — что-то пошло не так. Смотрите логи (раздел "Logs").

## ШАГ 7: Открываем сайт 🌐

### 7.1 Получаем ссылку

1. В Railway перейдите в раздел **"Settings"** (настройки)
2. Найдите блок **"Public Domain"** (публичный домен)
3. Там будет ссылка вида: `https://master-agent-serviceby-production.up.railway.app`
4. Нажмите на ссылку — откроется ваш агент!

### 7.2 Проверяем работу

1. Откроется страница с чатом
2. В левом верхнем углу должна быть зелёная плашка **"OpenRouter"**
3. Напишите в чат: `Привет!`
4. Агент должен ответить через 2-3 секунды

**Поздравляю! 🎉 Ваш Мастер-агент работает в интернете!**

## 🆘 Частые проблемы и решения

### Проблема: "Build failed" (сборка не удалась)

**Решение:**
1. Перейдите в раздел **"Deployments"**
2. Нажмите на красный деплой
3. Откройте вкладку **"Logs"**
4. Прокрутите вниз и найдите красный текст с ошибкой
5. Скопируйте ошибку и поищите решение в Google

**Частые причины:**
- Забыли добавить переменные окружения
- Проблема в коде (обратитесь к разработчику)

### Проблема: "Cannot GET /" или белый экран

**Решение:**
1. Проверьте что переменная `NODE_ENV` = `production`
2. Перезапустите деплой: Deployments → [ваш деплой] → "Redeploy"

### Проблема: Агент не отвечает на сообщения

**Решение:**
1. Проверьте что `OPENROUTER_API_KEY` правильный
2. Проверьте что на балансе OpenRouter есть деньги (нужно $1-2)
3. Посмотрите логи в Railway (раздел "Logs")

### Проблема: "Application Error"

**Решение:**
1. Перейдите в **"Variables"**
2. Убедитесь что все 3 переменные добавлены
3. Нажмите **"Redeploy"** (перезапустить)

## 💰 Оплата и лимиты

### Сколько это стоит?

**Railway:**
- Бесплатно: $5 в месяц
- Этого хватит для небольшого проекта
- Если лимит закончится — проект остановится

**OpenRouter:**
- Плата за использование ИИ (примерно $0.01-0.05 за запрос)
- Нужно пополнить баланс (минимум $5)
- Пополнить можно картой: https://openrouter.ai/credits

### Как пополнить OpenRouter

1. Зайдите на https://openrouter.ai/credits
2. Нажмите **"Add Credits"**
3. Введите сумму (минимум $5)
4. Введите данные карты
5. Нажмите **"Pay"**
