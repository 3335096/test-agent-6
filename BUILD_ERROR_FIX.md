# 🚨 Ошибка Build failed — Как исправить

Не паникуй! Это нормально — почти у всех бывает ошибка при первом деплое.

**Главное:** Нужно посмотреть логи (текст ошибки) и понять что не так.

---

## ШАГ 1: Посмотреть логи ошибки

### Как открыть логи:

1. В Railway перейдите в раздел **"Deployments"** (слева в меню)

2. Найдите ваш деплой с красной надписью **"Failed"**

3. **Нажмите на этот деплой** (на саму строчку)

4. Откроется страница с деталями деплоя

5. Нажмите на вкладку **"Logs"** (логи)

6. Прокрутите вниз — там будет красный текст с ошибкой

---

## ШАГ 2: Найти текст ошибки

В логах ищите:
- 🔴 **Красный текст**
- Слова: **"Error"**, **"Failed"**, **"Cannot"**, **"not found"**
- Последние 10-20 строк лога

**Скопируйте текст ошибки** (можно выделить и нажать Ctrl+C)

---

## 📋 Типичные ошибки и решения

---

### Ошибка 1: "Cannot find module" или "Module not found"

**Выглядит так:**
```
Error: Cannot find module 'express'
```

**Причина:** Не установлены зависимости (node_modules)

**Решение:**
1. Убедитесь что файл `package.json` есть в вашем репозитории на GitHub
2. Перейдите на GitHub → ваш репозиторий
3. Проверьте что там есть файл `package.json`
4. Если нет — загрузите его!

---

### Ошибка 2: "npm ERR! code ENOENT"

**Выглядит так:**
```
npm ERR! code ENOENT
npm ERR! syscall open
npm ERR! path /app/package.json
```

**Причина:** Railway не находит package.json

**Решение:**
1. На GitHub проверьте структуру файлов
2. Файл `package.json` должен быть в КОРНЕ репозитория
3. Не в папке, а сразу при открытии репозитория

**Правильная структура:**
```
master-agent-serviceby/
├── package.json      ← ДОЛЖЕН БЫТЬ ЗДЕСЬ
├── server.js
├── src/
│   └── App.tsx
└── ...
```

**Неправильная структура:**
```
master-agent-serviceby/
├── app/              ← ЛИШНЯЯ ПАПКА!
│   ├── package.json  ← НЕ ДОЛЖНО БЫТЬ В ПАПКЕ
│   └── server.js
```

---

### Ошибка 3: "sh: 1: tsc: not found"

**Выглядит так:**
```
sh: 1: tsc: not found
npm ERR! code ELIFECYCLE
npm ERR! errno 1
```

**Причина:** Railway не может найти TypeScript компилятор

**Решение:**

Вариант А — Убрать TypeScript сборку:
1. Откройте файл `package.json` на GitHub
2. Найдите строку:
   ```json
   "build": "tsc -b && vite build"
   ```
3. Измените на:
   ```json
   "build": "vite build"
   ```
4. Сохраните файл

Вариант Б — Добавить TypeScript в зависимости:
1. Откройте `package.json`
2. В разделе `dependencies` добавьте:
   ```json
   "typescript": "~5.9.3"
   ```

---

### Ошибка 4: "Cannot read file 'tsconfig.json'"

**Выглядит так:**
```
error TS5083: Cannot read file '/app/tsconfig.json'
```

**Причина:** Нет файла tsconfig.json

**Решение:**
1. Проверьте что файл `tsconfig.json` есть в репозитории
2. Если нет — загрузите его

---

### Ошибка 5: "src/App.tsx(123,45): error TS..."

**Выглядит так:**
```
src/App.tsx(123,45): error TS2345: Argument of type ...
```

**Причина:** Ошибка в коде TypeScript

**Решение:**
1. Посмотрите номер строки в ошибке (в скобках)
2. Откройте файл `src/App.tsx` на GitHub
3. Найдите эту строку
4. Исправьте ошибку или обратитесь к разработчику

---

### Ошибка 6: "No OPENROUTER_API_KEY provided"

**Выглядит так:**
```
Error: OPENROUTER_API_KEY не настроен
```

**Причина:** Не добавлена переменная окружения

**Решение:**
1. Перейдите в Railway → Variables
2. Добавьте переменную:
   - KEY: `OPENROUTER_API_KEY`
   - VALUE: ваш ключ `sk-or-v1-...`
3. Нажмите "Redeploy" (перезапустить деплой)

---

### Ошибка 7: "Cannot GET /" после успешного деплоя

**Выглядит так:**
- Статус деплоя: ✅ Success
- При открытии сайта: белый экран или "Cannot GET /"

**Причина:** Нет собранного фронтенда в папке dist

**Решение:**

1. Проверьте что папка `dist/` есть в репозитории
   - На GitHub откройте репозиторий
   - Проверьте есть ли папка `dist`

2. Если папки `dist` нет — нужно собрать фронтенд:
   
   **Вариант А — Собрать локально:**
   ```bash
   cd /mnt/okcomputer/output/app
   npm install
   npm run build
   ```
   Затем загрузите папку `dist` на GitHub
   
   **Вариант Б — Пусть Railway соберёт:**
   - Убедитесь что в `package.json` есть скрипт `"build": "vite build"`
   - Railway автоматически запустит сборку

3. Перезапустите деплой

## 🔧 Быстрая проверка файлов

### Откройте ваш репозиторий на GitHub и проверьте:

✅ **Должны быть эти файлы в КОРНЕ:**
- `package.json`
- `server.js`
- `railway.json`
- `Procfile`
- `tsconfig.json`
- `vite.config.ts`
- Папка `src/` с файлами
- Папка `dist/` (если собирали локально)

❌ **НЕ должно быть:**
- Папки `node_modules/` (очень большая, не нужна)
- Папки `app/` внутри репозитория (всё должно быть в корне)

## 🆘 Если ничего не помогает

### Вариант 1: Пересоздать проект

1. В Railway удалите проект:
   - Settings → Danger → Delete Project

2. Создайте заново:
   - New Project → Deploy from GitHub repo

### Вариант 2: Проверить через ZIP

1. Скачайте свой репозиторий с GitHub как ZIP
2. Распакуйте
3. Проверьте структуру папок

### Вариант 3: Написать в поддержку

Railway Discord: https://discord.gg/railway
Канал #support — опишите проблему и приложите логи
