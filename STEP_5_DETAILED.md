# ШАГ 5: Создаём проект на Railway — ПОДРОБНАЯ ИНСТРУКЦИЯ

Этот шаг самый важный! Здесь мы создадим вашего агента в интернете и настроим его.

---

## 📍 Где мы находимся

Вы должны быть:
1. ✅ Зарегистрированы на Railway (https://railway.app)
2. ✅ Вошли в свой аккаунт
3. ✅ Увидели панель управления Railway (Dashboard)

---

## ЧАСТЬ 1: Создание проекта

### 1.1 Начинаем создание

**Что видите:** Панель управления Railway с кнопками

**Что делать:**

1. Найдите большую кнопку **"+ New Project"** или просто **"New Project"**
   - Обычно она в центре экрана или в правом верхнем углу
   - Выглядит как зелёная или фиолетовая кнопка

2. **Нажмите на эту кнопку**

3. Откроется меню с вариантами:
   ```
   [ Deploy from GitHub repo ]  ← ВЫБИРАЕМ ЭТО
   [ Deploy from GitLab repo ]
   [ Deploy template ]
   [ Empty project ]
   ```

4. **Нажмите "Deploy from GitHub repo"**

---

### 1.2 Подключаем GitHub (первый раз)

Если вы первый раз используете Railway, нужно дать ему доступ к GitHub.

**Что видите:** Окно с кнопкой "Configure GitHub App"

**Что делать:**

1. **Нажмите кнопку "Configure GitHub App"**

2. Откроется новая вкладка GitHub с заголовком:
   **"Install Railway"**

3. Вы увидите два варианта:
   - ○ All repositories (все репозитории)
   - ● Only select repositories (только выбранные) ← ВЫБИРАЕМ ЭТО

4. **Нажмите на переключатель "Only select repositories"**

5. Появится выпадающий список **"Select repositories"**
   - Нажмите на это поле
   - В списке найдите свой репозиторий: `master-agent-serviceby`
   - **Кликните по нему**

6. Внизу страницы нажмите зелёную кнопку:
   **"Install & Authorize"**

7. Вас вернёт обратно в Railway

---

### 1.3 Выбираем репозиторий

**Что видите:** Список ваших репозиториев с GitHub

**Что делать:**

1. Найдите в списке: `ВАШ_НИК/master-agent-serviceby`
   - Например: `ivan123/master-agent-serviceby`

2. **Нажмите на этот репозиторий**

3. Railway начнёт анализировать код
   - Вы увидите анимацию загрузки
   - Ждите 10-30 секунд

---

### 1.4 Что происходит дальше

Railway автоматически:
- ✅ Определяет что это Node.js проект
- ✅ Находит файл `package.json`
- ✅ Готовится к сборке

**Вы увидите страницу с заголовком:**
"You're almost done! Add environment variables to get started"

(Вы почти закончили! Добавьте переменные окружения чтобы начать)

---

## ЧАСТЬ 2: Добавление переменных (САМОЕ ВАЖНОЕ!)

Это как настройки для вашего агента. Без них он не заработает.

---

### 2.1 Открываем настройки переменных

**Что видите:** Кнопка "Add Variables" или раздел Variables

**Что делать:**

1. **Нажмите кнопку "Add Variables"**
   - Или найдите в меню слева пункт "Variables" и нажмите

2. Откроется страница с таблицей:
   ```
   +----------------------------------+
   |  KEY        |  VALUE            |
   +----------------------------------+
   |  (пусто)    |  (пусто)          |
   +----------------------------------+
   |  [+ New Variable]               |
   +----------------------------------+
   ```

---

### 2.2 Добавляем ПЕРВУЮ переменную (API ключ)

Это самая важная переменная — ключ от ИИ.

**Шаг 1: Открываем форму добавления**

1. Нажмите кнопку **"+ New Variable"** (новая переменная)
   - Она зелёная или серая, с плюсиком

2. Появятся два поля ввода:
   ```
   KEY:   [                    ]
   VALUE: [                    ]
   ```

**Шаг 2: Заполняем поле KEY**

1. Кликните в поле **KEY**
2. Напишите (точно так, большими буквами):
   ```
   OPENROUTER_API_KEY
   ```
   ⚠️ Важно: без пробелов, с подчёркиваниями

**Шаг 3: Заполняем поле VALUE**

1. Кликните в поле **VALUE**
2. Вставьте ваш API ключ из Шага 1
   - Ключ выглядит так: `sk-or-v1-xxxxxxxxxxxxxxxx...`
   - Очень длинный, начинается на `sk-or-v1-`
3. **Проверьте что весь ключ вставился полностью!**

**Шаг 4: Сохраняем**

1. Нажмите кнопку **"Add"** или **"Save"**
2. Переменная появится в таблице:
   ```
   +----------------------------------+
   |  OPENROUTER_API_KEY | sk-or...  |
   +----------------------------------+
   ```

---

### 2.3 Добавляем ВТОРУЮ переменную (Модель ИИ)

Эта переменная говорит какой ИИ использовать.

**Шаг 1: Открываем форму**

1. Снова нажмите **"+ New Variable"**

**Шаг 2: Заполняем**

| Поле | Что вписать |
|------|-------------|
| **KEY** | `OPENROUTER_MODEL` |
| **VALUE** | `anthropic/claude-3.5-sonnet` |

**Шаг 3: Сохраняем**

1. Нажмите **"Add"**

---

### 2.4 Добавляем ТРЕТЬЮ переменную (Режим работы)

Эта переменная говорит что мы запускаем production версию.

**Шаг 1: Открываем форму**

1. Нажмите **"+ New Variable"**

**Шаг 2: Заполняем**

| Поле | Что вписать |
|------|-------------|
| **KEY** | `NODE_ENV` |
| **VALUE** | `production` |

**Шаг 3: Сохраняем**

1. Нажмите **"Add"**

---

### 2.5 Проверяем результат

**Что должно быть в таблице:**

```
+--------------------------------------------------+
|  KEY                  |  VALUE                    |
+--------------------------------------------------+
|  OPENROUTER_API_KEY   |  sk-or-v1-ваш-ключ...     |
|  OPENROUTER_MODEL     |  anthropic/claude-3.5...  |
|  NODE_ENV             |  production               |
+--------------------------------------------------+
```

**Все 3 переменные добавлены? Отлично!**

---

## ЧАСТЬ 3: Запускаем деплой

### 3.1 Находим кнопку Deploy

После добавления переменных Railway покажет кнопку для запуска.

**Варианты где может быть кнопка:**

1. **Вверху страницы Variables:**
   - Большая зелёная/фиолетовая кнопка **"Deploy"**

2. **Или в меню слева:**
   - Нажмите **"Deployments"**
   - Там будет кнопка **"Deploy Now"**

### 3.2 Нажимаем Deploy

1. **Нажмите кнопку "Deploy"**

2. Railway начнёт процесс:
   ```
   Building...     ← Собирает проект
   Deploying...    ← Разворачивает
   ```

3. **Ждите 3-5 минут**
   - Не закрывайте страницу
   - Можно смотреть логи (откроются автоматически)

---

## ЧАСТЬ 4: Проверяем что всё работает

### 4.1 Смотрим статус деплоя

1. Перейдите в раздел **"Deployments"** (слева в меню)

2. Вы увидите список деплоев:
   ```
   +------------------------------------------+
   |  Deployment          |  Status           |
   +------------------------------------------+
   |  #1 (только что)     |  🟢 Success       |  ← ХОРОШО
   |  #2 (если были)      |  ⚪ Stopped       |
   +------------------------------------------+
   ```

3. **Если статус "Success" (зелёная галочка)** — всё отлично!

4. **Если статус "Failed" (красный крестик)** — смотрите логи (раздел "Logs")

---

### 4.2 Получаем ссылку на сайт

1. Перейдите в раздел **"Settings"** (Настройки) — слева в меню

2. Прокрутите вниз до раздела **"Public Domain"**

3. Там будет ссылка вида:
   ```
   https://master-agent-serviceby-production.up.railway.app
   ```
   или
   ```
   https://master-agent-abc123.railway.app
   ```

4. **Нажмите на эту ссылку** — откроется ваш агент!

---

## 🆘 Если что-то пошло не так

### Проблема: Не вижу кнопку "Deploy"

**Решение:**
1. Убедитесь что добавили все 3 переменные
2. Перейдите в раздел "Deployments" слева
3. Там должна быть кнопка "Deploy Now"

### Проблема: Ошибка "Build failed"

**Что делать:**
1. Нажмите на красный деплой в списке
2. Откройте вкладку **"Logs"**
3. Прокрутите вниз до красного текста
4. Скопируйте ошибку
5. Скорее всего:
   - Забыли добавить переменные → добавьте
   - Неправильный формат ключа → проверьте ключ

### Проблема: Сайт открывается, но агент не отвечает

**Что делать:**
1. Проверьте что `OPENROUTER_API_KEY` правильный
2. Проверьте баланс на OpenRouter (нужно $1-2)
3. Посмотрите логи в Railway
