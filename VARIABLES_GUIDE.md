# 📍 Где добавить переменные в Railway — Подробная инструкция

---

## ШАГ 1: Открываем проект

1. Зайдите на https://railway.app
2. Войдите в свой аккаунт
3. Нажмите на название вашего проекта (например, "master-agent-serviceby")

---

## ШАГ 2: Находим раздел Variables

**Способ 1 — Через меню слева:**

1. Слева есть вертикальное меню
2. Найдите пункт **"Variables"** (иконка 🔧 или шестерёнка)
3. **Нажмите на него**

**Способ 2 — Через верхнее меню:**

1. Вверху страницы есть вкладки
2. Найдите вкладку **"Variables"**
3. **Нажмите на неё**

---

## ШАГ 3: Что вы увидите

Откроется страница с таблицей:

```
+------------------------------------------+
│  KEY (название)  │  VALUE (значение)     │
+------------------------------------------+
│  (пусто)         │  (пусто)              │
+------------------------------------------+
│                                            │
│  [+ New Variable]  ← ЗЕЛЁНАЯ КНОПКА       │
│                                            │
+------------------------------------------+
```

---

## ШАГ 4: Добавляем первую переменную

### 4.1 Нажимаем кнопку

Нажмите зелёную кнопку **"+ New Variable"**

### 4.2 Заполняем поля

Появятся два поля:

**Поле KEY (слева):**
```
┌─────────────────────────────────┐
│  OPENROUTER_API_KEY             │
└─────────────────────────────────┘
```
- Пишем БОЛЬШИМИ буквами: `OPENROUTER_API_KEY`
- Без пробелов, с подчёркиваниями

**Поле VALUE (справа):**
```
┌─────────────────────────────────┐
│  sk-or-v1-ваш-длинный-ключ...   │
└─────────────────────────────────┘
```
- Вставляете ваш API ключ из OpenRouter
- Ключ начинается на `sk-or-v1-`

### 4.3 Сохраняем

Нажмите кнопку **"Add"** или **"Save"**

---

## ШАГ 5: Добавляем вторую переменную

Снова нажмите **"+ New Variable"**

| Поле | Что вписать |
|------|-------------|
| **KEY** | `OPENROUTER_MODEL` |
| **VALUE** | `anthropic/claude-3.5-sonnet` |

Нажмите **"Add"**

---

## ШАГ 6: Добавляем третью переменную

Снова нажмите **"+ New Variable"**

| Поле | Что вписать |
|------|-------------|
| **KEY** | `NODE_ENV` |
| **VALUE** | `production` |

Нажмите **"Add"**

---

## ШАГ 7: Проверяем результат

В таблице должно быть ТРИ строки:

```
+--------------------------------------------------+
│  KEY                  │  VALUE                    │
+--------------------------------------------------+
│  OPENROUTER_API_KEY   │  sk-or-v1-...             │
│  OPENROUTER_MODEL     │  anthropic/claude-3.5...  │
│  NODE_ENV             │  production               │
+--------------------------------------------------+
```

**Все три переменные добавлены? Отлично!**

---

## ШАГ 8: Перезапускаем деплой

После добавления переменных Railway автоматически перезапустит проект.

**Если НЕ перезапустил:**

1. Перейдите в раздел **"Deployments"** (слева в меню)
2. Найдите последний деплой
3. Нажмите на три точки (⋯)
4. Выберите **"Redeploy"** (перезапустить)

## ⚠️ Частые ошибки

### Ошибка 1: Переменная не сохранилась

**Причина:** Забыли нажать "Add"

**Решение:** Нажмите "Add" после заполнения каждой переменной

### Ошибка 2: Опечатка в названии

**Неправильно:**
- `OpenRouter_Api_Key` ← маленькие буквы
- `OPENROUTER-API-KEY` ← дефисы вместо подчёркиваний
- `openrouter api key` ← пробелы

**Правильно:**
- `OPENROUTER_API_KEY` ← БОЛЬШИЕ буквы, подчёркивания

### Ошибка 3: Ключ обрезался

**Причина:** API ключ очень длинный, вставился не полностью

**Решение:**
1. Удалите переменную
2. Добавьте заново
3. Проверьте что ключ начинается на `sk-or-v1-` и заканчивается на буквы/цифры
